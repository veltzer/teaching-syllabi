# Real Time Design Patterns

## Credits
Mark Veltzer [mark.veltzer@gmail.com](mailto:mark.veltzer@gmail.com)

## Description
This course is about how to design the internals of a Real Time system correctly based on tried and proven
design patterns prevalent in the industry.

## Duration
2 days / 16 hours

## Intended Audience
* Software architects that do real time system designs
* C and C++ programmers who would like to design real time systems

## Prerequisites
* C programming
* Embedded programming is an advantage

## Outline
* Introduction to Real-Time Systems and Design Patterns
    * Characteristics of real-time systems
    * Review of traditional design patterns
    * Introduction to real-time specific patterns
* Concurrency Patterns
    * Active Object Pattern
    * Monitor Object Pattern
    * Thread-Safe Interface Pattern
* Resource Management Patterns
    * Resource Pool Pattern
    * Critical Section Pattern
    * Strategized Locking Pattern
* Distribution Patterns
    * Broker Pattern
    * Proactor Pattern
    * Reactor Pattern
* Event Handling Patterns
    * Observer Pattern in real-time context
    * Publish-Subscribe Pattern
    * Event-based Asynchronous Pattern
* Timing Patterns
    * Deadline Mechanism Pattern
    * Temporal Coherence Pattern
    * Watchdog Pattern
* Fault Tolerance Patterns
    * Redundancy Pattern
    * Checkpoint Pattern
    * Recovery Block Pattern

## References
* [Bruce Powel Douglass, Ph.D, Real-Time Design Patterns](http://www.uml.org.cn/UMLApplication/pdf/rtpatterns.pdf)
