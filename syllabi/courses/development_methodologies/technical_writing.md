# Technical Writing

## Description
Clear and effective technical writing is an essential skill for software professionals.
This course teaches the fundamentals of writing documentation that is accurate, concise,
and useful to its intended audience. Topics range from everyday documents like READMEs
and `API` references to architecture decision records and runbooks. Students will learn
modern tooling, documentation-as-code practices, and strategies for keeping documentation
maintainable over time.

## Duration
16 hours / 2 days

## Intended Audience
* Developers who write documentation alongside code.
* Technical leads responsible for team documentation standards.
* Managers who want to improve documentation culture in their organizations.

## Prerequisites
* Basic familiarity with software development workflows.
* Working knowledge of `Markdown`.

## Objectives
* understand the core concepts and principles of technical writing
* gain practical knowledge of audience analysis and document planning
* gain practical knowledge of documentation tooling and workflows
* gain practical knowledge of writing clear `API` and reference documentation
