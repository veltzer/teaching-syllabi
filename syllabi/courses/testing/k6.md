# `k6`

## Description
This course introduces `k6`, the modern open-source load testing tool designed for developer workflows, and covers everything needed to design, execute, and interpret realistic load tests. Students will learn to write test scripts in `JavaScript`, model complex user journeys with virtual users and scenarios, and define pass/fail criteria using thresholds and checks. Advanced topics include `HTTP` and `WebSocket` protocol testing, test data parameterisation, distributed testing with `k6 Cloud`, and integrating load tests into `CI/CD` pipelines. By the end, participants will be able to measure and validate system performance with confidence.

## Duration
16 hours / 2 days

## Intended Audience
* Backend and full-stack developers who own service performance
* QA engineers responsible for non-functional testing
* `DevOps` and platform engineers integrating performance tests into delivery pipelines

## Prerequisites
* Working knowledge of `JavaScript` (`ES6`+)
* Basic understanding of `HTTP` and `REST` APIs
* Familiarity with command-line tools and basic `CI/CD` concepts

## Objectives
* Explain load testing concepts and apply the appropriate test type for each scenario
* Install and configure `k6` in local and CI environments
* Write `k6` test scripts using virtual users, stages, and scenarios
* Define thresholds and checks to express pass/fail acceptance criteria
* Test `HTTP` endpoints, WebSockets, and `gRPC` services
* Parameterise tests with external data files and environment variables
* Execute distributed load tests using `k6 Cloud`
* Integrate `k6` into `CI/CD` pipelines and analyse output metrics

## Outline
<!-- chapter: introduction-to-load-testing, duration: 1h -->
* Introduction to Load Testing
    * Why performance matters: user experience and business impact
    * Categories of performance tests: load, stress, spike, soak, breakpoint
    * Key metrics: response time, throughput, error rate, concurrency
    * Performance testing strategy and when to run each test type
    * Common performance anti-patterns and how tests reveal them
<!-- chapter: k6-architecture-and-installation, duration: 1h -->
* `k6` Architecture and Installation
    * How `k6` works: the `JavaScript` runtime and Go execution engine
    * Installing `k6` on `Linux`, `macOS`, and `Windows`
    * Running `k6` in `Docker`
    * Available output formats: stdout, `JSON`, `InfluxDB`, `Prometheus`, `Datadog`
    * Overview of the `k6` extension ecosystem
<!-- chapter: writing-your-first-test-script, duration: 2h -->
* Writing Your First Test Script
    * Anatomy of a `k6` script: `default` function and options
    * Making `HTTP` requests with `http.get` and `http.post`
    * The check function for validating responses
    * Setting virtual users (`vus`) and duration
    * Using `sleep` to simulate user think time
    * Organising scripts into modules and helper functions
<!-- chapter: virtual-users-and-scenarios, duration: 2h -->
* Virtual Users and Scenarios
    * How virtual users model real user concurrency
    * Ramping VU profiles: ramp-up, steady-state, ramp-down
    * Defining stages with `stages` configuration
    * Introduction to the `scenarios` `API`
    * Executor types: `constant-vus`, `ramping-vus`, `constant-arrival-rate`, `ramping-arrival-rate`
    * Combining multiple scenarios in one test run
    * Using `exec` to point scenarios at specific functions
<!-- chapter: thresholds-and-checks, duration: 2h -->
* Thresholds and Checks
    * Defining `thresholds` for built-in and custom metrics
    * Threshold expressions: `p(95)<200`, `rate<0.01`
    * Aborting tests early when thresholds are exceeded
    * The check function vs assertions
    * Creating custom metrics: `Counter`, Gauge, Rate, `Trend`
    * Group-level checks and per-group metrics
<!-- chapter: http-and-websocket-testing, duration: 2h -->
* `HTTP` and `WebSocket` Testing
    * Full `HTTP` request lifecycle in `k6`: headers, body, cookies
    * Handling sessions and authentication: bearer tokens, cookies, `OAuth` flows
    * `HTTP` batch requests for parallel calls
    * Parsing and asserting `JSON` response bodies
    * `WebSocket` testing with the `k6/experimental/websockets` `API`
    * `gRPC` testing with `k6/net/grpc`
    * Handling redirects, timeouts, and retries
<!-- chapter: test-data-and-parameterization, duration: 2h -->
* Test Data and Parameterisation
    * Reading data files with `SharedArray` for efficiency
    * `CSV` and `JSON` test data formats
    * Using environment variables for configuration
    * Generating dynamic data per iteration
    * Unique user simulation with data pools
    * Handling stateful workflows: login tokens and session `IDs`
<!-- chapter: k6-cloud-and-distributed-testing, duration: 2h -->
* `k6 Cloud` and Distributed Testing
    * `When` local execution is not enough
    * `k6 Cloud` architecture and execution model
    * Running cloud tests with `k6 cloud run`
    * Selecting geographic load zones
    * Real-time streaming results in the cloud dashboard
    * Comparing test runs and tracking trends over time
    * On-premise distributed execution with `k6 operator` on `Kubernetes`
<!-- chapter: integrating-k6-with-cicd, duration: 1h -->
* Integrating `k6` with `CI/CD`
    * Running `k6` in `GitHub Actions`, `GitLab CI`, and `Jenkins`
    * Passing environment-specific configuration at run time
    * Failing CI builds based on threshold violations
    * Storing and archiving test results as artefacts
    * Triggering performance gates before production releases
<!-- chapter: analyzing-results, duration: 1h -->
* Analyzing Results
    * Reading the default end-of-test summary
    * Exporting metrics to `InfluxDB` and visualising in `Grafana`
    * `Prometheus` remote write integration
    * Correlating `k6` metrics with backend traces and logs
    * Identifying bottlenecks from percentile distributions
    * Iterating on test scripts based on findings
