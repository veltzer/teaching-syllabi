# `UML`

## Description
This course provides comprehensive training in the Unified Modeling Language (`UML`) for software design and architecture. Participants will learn to create and interpret the most important `UML` diagram types, from requirements capture through detailed design. The course covers both structural and behavioral diagrams, with emphasis on practical modeling techniques that improve communication between stakeholders and development teams.

The course focuses on `UML` 2.x notation and conventions, modeling best practices, and the effective use of `UML` tools in modern software projects.

## Duration
16 hours / 2 days

## Intended Audience
* Software developers involved in system design
* Software architects creating technical specifications
* Team leads communicating design decisions
* Analysts translating requirements into design models

## Prerequisites
* Experience with software development
* Basic understanding of object-oriented programming concepts
* Familiarity with software development lifecycle

## Objectives
* Understand the purpose and scope of `UML` in software development
* Create and interpret use case diagrams for requirements modeling
* Design class diagrams that accurately represent system structure
* Model dynamic behavior with sequence and activity diagrams
* Represent system states using state machine diagrams
* Document system architecture with component and deployment diagrams
* Apply modeling best practices for clarity and maintainability
