# `Java` `Design Patterns`

## Description
This course covers the classic Gang of Four `design patterns` and their practical application
in `Java`. Participants will learn how to apply creational, structural, and behavioral patterns,
understand `SOLID` principles, recognize anti-patterns, and refactor existing code toward
well-known patterns. The course emphasizes real-world examples and enterprise patterns
commonly used in `Java` applications.

## Duration
24 hours / 3 days

## Intended Audience
* `Java` developers who want to write more maintainable and extensible code.
* Software architects looking to deepen their understanding of `design patterns` in `Java`.

## Prerequisites
* Working knowledge of `Java` programming.
* Familiarity with object-oriented programming concepts.

## Objectives
* Participants will gain a thorough understanding of
    * `SOLID` principles and their role in software design.
    * GoF creational, structural, and behavioral `design patterns`.
    * `When` and how to apply each pattern in real-world `Java` projects.
    * Common anti-patterns and how to avoid them.
    * Refactoring techniques to evolve code toward patterns.
