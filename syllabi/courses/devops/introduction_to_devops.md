# Introduction to `DevOps`

## Credits
Mark Veltzer [mark.veltzer@gmail.com](mailto:mark.veltzer@gmail.com)

## Description
This 40-hour course introduces students to the principles and practices of `DevOps`. Students will learn about the culture, methodologies, and tools used in `DevOps` to improve collaboration between development and operations teams, increase efficiency, and deliver high-quality software faster.

## Duration
40 hours / 5 days

## Intended Audience
* Software developers looking to expand their knowledge into operations and `automation`
* `IT` operations professionals interested in learning about modern software development practices
* System administrators wanting to update their skills for `cloud` and `DevOps` environments
* Recent graduates in computer science or related fields seeking to enter the `DevOps` field
* Project managers and team leads who want to understand and implement `DevOps` practices
* Any IT professional interested in learning about `DevOps` principles and practices

## Prerequisites
* Basic understanding of software development concepts
* Familiarity with command-line interfaces
* Basic knowledge of operating systems (`Linux`/`Unix` preferred)
* Some exposure to cloud computing concepts (helpful but not required)

## Objectives
* Understand the core principles and benefits of `DevOps`
* Implement basic Continuous Integration and Continuous Deployment (`CI/CD`) pipelines
* Use version control systems effectively
* Apply Infrastructure as Code (`IaC`) concepts
* Implement basic monitoring and logging solutions
* Understand containerization and orchestration fundamentals
* Apply `DevOps` practices to improve software development lifecycle

## Outline
* What is `DevOps`?
    * Definition and history
    * `DevOps` culture and principles
    * Benefits and challenges
* `DevOps` Practices and Methodologies
    * Agile and Lean principles in `DevOps`
    * Continuous Integration and Continuous Deployment (`CI/CD`)
    * Infrastructure as Code (`IaC`)
* Version Control with `Git`
    * Basic `Git` commands and workflow
    * Branching strategies
    * Collaborative development with `Git`
* Continuous Integration
    * CI concepts and best practices
    * Setting up a CI server (e.g., `Jenkins`, GitLab CI)
    * Creating and running automated tests
* Continuous Deployment
    * CD concepts and best practices
    * Deployment strategies
    * Implementing a basic CD pipeline
* Infrastructure as Code
    * `IaC` concepts and tools
    * Introduction to `Terraform`
    * Creating and managing infrastructure with `Terraform`
* Introduction to Containerization
    * Container concepts and benefits
    * `Docker` basics
    * Creating and managing `Docker` images and containers
* Container Orchestration with `Kubernetes`
    * `Kubernetes` architecture and components
    * Deploying applications on `Kubernetes`
    * Managing and scaling applications in `Kubernetes`
* Monitoring and Logging
    * Importance of monitoring in `DevOps`
    * Basic monitoring concepts and tools
    * Log management and analysis
* Security in `DevOps` (DevSecOps)
    * Integrating security into the `DevOps` lifecycle
    * Security best practices in `CI/CD`
* Cloud Platforms and `DevOps`
    * Overview of cloud services (`AWS`, `Azure`, `GCP`)
    * `DevOps` practices in cloud environments

## Textbooks and Resources
* `The Phoenix Project` by Gene Kim, Kevin Behr, and George Spafford
* `DevOps Handbook` by Gene Kim, Jez Humble, Patrick Debois, and John Willis
* `Continuous Delivery` by Jez Humble and David Farley
