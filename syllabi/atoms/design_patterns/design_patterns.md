# Design patterns

## Credits
Mark Veltzer [mark.veltzer@gmail.com](mailto:mark.veltzer@gmail.com)

## Description
A "Design pattern" describes a solution to a problem which occurs over and over again
in software development. This solution is given a name, a rationale, advantages and
disadvantages compared to other solution and can be used a million times over,
even with modifications, without having to do the heavy lifting of coming up with the
solution from scratch.

In this course you will get a deep understanding of the most common object oriented
design patterns. You will understand how to use them in different projects.
You will know how using design patterns help you to receive higher maintainability,
re-usability and extensibility.

This course is usually taught tied to a specific programming languages, usually
an object oriented one like `Java`, `C#`, `Python` or C++.

## Duration
24 hours / 3 days

## Intended Audience
* Software developers with experience in an object oriented language

## Prerequisites
* At least one year experience in programming an object oriented language.
* Understanding of the following:
    * abstract data type
    * encapsulation
    * overriding
    * overloading
    * inheritance
    * polymorphism
    * abstract classes
    * virtual methods

## Objectives
* you will get better and more ideas on how to model a particular technical problem
* you will be able to reuse and apply design ideas which are proven to be effective
* you will get to a more appropriate granularity of your classes that model a particular domain
