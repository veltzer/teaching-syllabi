Introduction to Architecting

Credits

Mark Veltzer mark.veltzer@gmail.com

Description

This one-day course introduces participants to the fundamental concepts and practices of software architecture. Attendees will learn the role of an architect, key architectural patterns, and how to make informed design decisions.

Duration

8 hours / 1 day

Intended Audience

  • Those who wish to participate in more professional architecting courses like
    • Architecting in the close
    • Architecting on AWS/Azure/GCP
    • Large scale architecting

Prerequisites

  • Tech experience of some sort
  • Basic understanding of software development concepts
  • Familiarity with at least one programming language

Objectives

  • Define software architecture and its importance
  • Identify key responsibilities of a software architect
  • Understand common architectural styles and patterns
  • Apply basic principles of architectural decision-making
  • Recognize the importance of non-functional requirements in architecture

Outline

  • Fundamentals of Software Architecture
    • Definition and importance of software architecture
    • The role of a software architect
    • Architecture vs. design
  • Architectural Styles and Patterns
    • Monolithic architecture
    • Microservices architecture
    • Layered architecture
    • Event-driven architecture
    • Domain-driven design
  • Architectural Decision Making
    • Architectural drivers
    • Trade-off analysis
    • Documentation of architectural decisions (ADR - Architecture Decision Record)
  • Non-Functional Requirements and Quality Attributes
    • Performance
    • Scalability
    • Security
    • Maintainability
    • Reliability
  • Case Study
    • Applying concepts to a real-world scenario
    • Group discussion and analysis
